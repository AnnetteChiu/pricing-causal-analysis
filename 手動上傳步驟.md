# 手動上傳到 GitHub 的詳細步驟

## 🎯 方法一：通過 GitHub 網頁界面上傳

### 步驟 1：創建倉庫
1. 訪問 https://github.com/new
2. 填寫：
   - Repository name: `pricing-causal-analysis`
   - Description: `因果推論在定價策略上的應用`
   - 選擇 Public
   - **不要**勾選任何初始化選項
3. 點擊 "Create repository"

### 步驟 2：上傳文件
1. 在新創建的倉庫頁面，點擊 "uploading an existing file"
2. 將以下文件拖拽到上傳區域：

**核心文件**：
- `pricing_causal_analysis.py`
- `advanced_pricing_methods.py`
- `demo_analysis.py`
- `run_analysis.py`
- `requirements.txt`

**文檔文件**：
- `README.md`
- `使用指南.md`
- `項目總覽.md`
- `GitHub部署指南.md`

**配置文件**：
- `setup.py`
- `LICENSE`
- `.gitignore`
- `CONTRIBUTING.md`
- `CHANGELOG.md`

3. 在 "Commit changes" 部分填寫：
   - Commit message: `🎉 Initial commit: 因果推論在定價策略上的應用`
4. 點擊 "Commit changes"

### 步驟 3：創建文件夾結構
由於 GitHub 網頁界面不能直接創建文件夾，需要：

1. 點擊 "Create new file"
2. 輸入 `.github/workflows/ci.yml`
3. 複製 CI/CD 配置內容
4. 提交

重複以上步驟創建：
- `.github/ISSUE_TEMPLATE/bug_report.md`
- `.github/ISSUE_TEMPLATE/feature_request.md`
- `tests/__init__.py`
- `tests/test_basic_analysis.py`

## 🎯 方法二：使用命令行（推薦）

如果你已經在本地有了所有文件，使用命令行更簡單：

### 前提條件
確保已經配置了 Git 和 GitHub 認證：
```bash
# 配置 Git（如果還沒配置）
git config --global user.name "Annette Chiu"
git config --global user.email "your-email@example.com"

# 檢查 GitHub 認證
gh auth status  # 如果安裝了 GitHub CLI
```

### 執行步驟
1. **創建 GitHub 倉庫**（在 GitHub 網站上）
2. **推送代碼**：
```bash
# 添加遠程倉庫
git remote add origin https://github.com/AnnetteChiu/pricing-causal-analysis.git

# 推送代碼
git push -u origin main
```

## 🎯 方法三：使用 GitHub CLI（最簡單）

如果安裝了 GitHub CLI：

```bash
# 創建倉庫並推送（一步完成）
gh repo create pricing-causal-analysis --public --description "因果推論在定價策略上的應用" --push
```

## ❓ 常見問題

**Q: 推送時要求認證怎麼辦？**
A: 需要設置 GitHub 個人訪問令牌（Personal Access Token）

**Q: 文件太大無法上傳？**
A: 檢查 .gitignore 是否正確配置，排除不必要的文件

**Q: 中文文件名顯示異常？**
A: 這是正常的，GitHub 會正確處理中文文件名
